> 最后更新：2026-07-10

## 一、产品定位

一个纯前端单页应用，用户只需输入自己各厂商的 API Key，即可在一个界面中对接 15+ 家国内外 AI 厂商的所有主流模型，支持单模型对话、多模型并行对比、可配置轮数的模型辩论，并集成代码执行、文件解析、语音输入/播报等增强功能。

### 2.2 三种对话模式

#### 💬 单模型模式
- 选择 1 个模型直接对话
- 流式输出 + 思考链展示

#### 📋 多模型并行模式
- 选择 2~∞ 个模型
- 同一问题同时发送，并排对比结果
- 网格布局展示各模型回答

#### ⚔️ 辩论模式
- 选择 2~∞ 个模型参与辩论
- **辩论轮数无上限**，用户通过精美滑条控件设置（1~99 轮，默认 3）
- 滑条 UI 设计：
  - 圆角轨道 + 渐变填充
  - 拖拽圆形把手
  - 左侧数字实时显示
  - 支持直接输入数字
  - 预设快捷按钮：3轮 / 5轮 / 10轮 / 20轮
- 每轮自动注入前轮所有模型观点作为上下文
- 最后一轮各模型总结陈词

### 2.3 流式输出与思考链

#### 打字机效果
- 使用 ReadableStream 逐字渲染
- 流式过程中 Markdown 实时解析（不中断渲染）
- 未闭合的代码块/表格在流式过程中也能正确显示

#### 思考链展示（Deep Thinking）
- 识别模型返回的 thinking/reasoning_content 字段
- 思考过程默认折叠，显示"🧠 思考过程（点击展开）"
- 展开后显示完整思考内容，带 Markdown 渲染
- 思考中状态：脉冲动画 + "思考中..."
- 支持的模型：
  - Anthropic Claude：`thinking` content block
  - DeepSeek R1：`reasoning_content` 字段
  - OpenAI o1/o3：reasoning tokens
  - MiMo V2.5-Pro：reasoning 字段

### 2.4 富文本渲染引擎

对标 Kimi / ChatGPT / Claude 等主流 AI 界面的输出质量。

#### Markdown 完整支持
- 标题（h1-h6）带分级样式
- 粗体、斜体、删除线
- 有序/无序列表、嵌套列表
- 引用块（左侧渐变边框）
- 水平分割线
- 链接（可点击，新窗口打开）
- 图片（自适应宽度，圆角）

#### 代码块
- 语法高亮（highlight.js，100+ 语言）
- 代码块右上角"复制"按钮
- 代码块顶部显示语言标签
- 等宽字体 + 行号（可选）

#### 表格
- 精美样式：斑马纹、hover 高亮、固定表头
- 响应式：移动端横向滚动
- Markdown 表格自动转换

#### 数学公式
- LaTeX 行内公式：`$...$`
- LaTeX 块级公式：`$$...$$`
- 使用 KaTeX 渲染，支持所有常用数学符号

#### Mermaid 图表
- 流程图（flowchart）
- 时序图（sequence diagram）
- 甘特图（gantt）
- 思维导图（mindmap）
- 自动检测 ` ```mermaid ` 代码块并渲染

#### 流式安全渲染
- 流式输出过程中，不完整的 Markdown 标签不渲染（防止闪烁）
- 代码块未闭合时保持原始文本
- 表格行未结束时暂不渲染

### 2.5 Python 代码执行（对标 Kimi）

#### 功能说明
- 模型生成的 Python 代码块自动检测
- 代码块右下角显示 ▶ "运行" 按钮
- 点击后在浏览器内执行 Python 代码

#### 执行方案
- **Pyodide**（WebAssembly 版 CPython）：
  - 纯浏览器端执行，无需后端
  - 支持 numpy、pandas、matplotlib、scipy 等科学计算库
  - 代码输出显示在代码块下方（控制台输出 / 图表 / 表格）
- 支持的功能：
  - `print()` 输出
  - matplotlib 图表渲染（输出为图片）
  - pandas DataFrame 渲染（输出为表格）
  - 数学计算、字符串处理
  - 文件读写（内存文件系统）

#### 输出展示
- 文本输出：等宽字体代码块
- 图表输出：内嵌图片
- 表格输出：精美 HTML 表格
- 错误信息：红色高亮

### 2.6 文件/图片上传

#### 上传方式
- 输入框左侧 📎 按钮点击上传
- 支持拖拽文件到输入区域
- 支持粘贴截图（Ctrl+V）

#### 支持的文件类型

| 类型 | 格式 | 处理方式 |
|------|------|----------|
| 图片 | jpg, png, webp, gif | 转 base64，发给视觉模型 |
| 文档 | pdf, docx, txt, md | 读取文本内容，附在消息中 |
| 数据 | csv, json, xlsx | 解析为结构化数据 |
| 代码 | py, js, ts, java, c, cpp, go, rs, html, css, sql | 读取源码 |
| 音频 | mp3, wav, m4a, webm | 用 MiMo ASR 转文字 |

#### 大小限制
- 单文件最大 20MB
- 图片自动压缩到 2048px 以内
- 文本文件截取前 100KB

#### 智能路由
- 上传图片 → 只有带 `👁️视觉` 标签的模型会将其作为图片输入
- 上传文档 → 只有带 `📎文件` 标签的模型会附上文件内容
- 不支持的模型 → 附件以纯文本描述形式附在消息中

### 2.7 语音输入

#### 优先级：本地优先，云端兜底

**方案 1：Web Speech API（浏览器内置）— 首选**
- 优点：零配置、零延迟、免费、无需 API Key
- 缺点：仅 Chrome/Edge 支持、需联网（Google 服务器）
- 使用 `SpeechRecognition` 接口
- 设置 `lang: 'zh-CN'` 识别中文
- 安静环境下准确率 95%+
- 实时显示识别中间结果

**方案 2：Whisper.cpp WASM（本地离线）— 备选**
- 完全本地运行，无需联网
- 基于 OpenAI Whisper 模型的 C++ WebAssembly 移植
- 模型文件 ~40MB（tiny）到 ~150MB（base），首次加载后缓存
- 中文识别效果好
- 缺点：首次加载慢、需要下载模型文件

**方案 3：小米 MiMo-V2.5-ASR — 兜底**
- 当本地方案不可用时使用
- 需要配置 MiMo API Key
- 调用 `https://api.mimo.mi.com/v1/audio/transcriptions`
- 支持中英双语 + 方言
- 费用：¥0.5/小时

#### 语音输入 UI
- 输入框左侧 🎤 按钮
- 点击开始录音（按钮变红 + 脉冲动画）
- 再次点击停止录音
- 显示当前使用的是哪种识别方案
- 识别结果自动填入输入框
- 用户可编辑后发送

### 2.8 文字转语音播报（TTS）

#### 功能
- 每条 AI 回复右下角显示 🔊 播报按钮
- 点击后朗读该条回复内容

#### 方案
- **首选：Web Speech Synthesis API（浏览器内置）**
  - 零配置，免费
  - 支持中英文
  - 可选语音、语速、音调
  - Chrome/Edge/Safari 均支持
- **高级：MiMo-V2.5-TTS**（如已配置 MiMo Key）
  - 高质量语音合成
  - 支持多音色、情绪控制
  - 费用：限时免费

#### 播报 UI
- 点击 🔊 开始播报，按钮变为 ⏸ 暂停
- 播报进度条
- 可选择语速：0.5x / 1x / 1.5x / 2x

### 2.9 API Key 验证

#### 验证时机
- 每次**选择模型**或**发送消息**前，自动验证对应厂商的 Key 是否有效
- API Keys 页面保存 Key 后立即验证

#### 验证方式
- 向各厂商发送一个最小化请求（1 token），检查返回状态
- 成功：绿勾 ✅ + "已验证"
- 失败：红叉 ❌ + 错误信息（如 "401 Unauthorized"、"余额不足"等）
- 验证中：⏳ 旋转动画

#### 验证 UI
- API Keys 页面每个 Key 输入框右侧显示验证状态
- 模型选择器中，已验证的厂商显示绿点，失败显示红点
- 发送消息时如果 Key 无效，阻止发送并 toast 提示

## 四、技术方案

### 依赖库（CDN）

### 语音方案

```
语音输入流程:
┌─────────────┐    失败    ┌──────────────┐    失败    ┌──────────────┐
│ Web Speech  │ ────────→ │ whisper.cpp  │ ────────→ │ MiMo ASR    │
│ API (免费)   │           │ WASM (本地)   │           │ (云端, ¥0.5/h)│
└─────────────┘           └──────────────┘           └──────────────┘
  Chrome/Edge               首次加载~40MB               需 MiMo Key
  准确率95%+                完全离线                    中英+方言
```

### 数据持久化

| 数据 | 存储位置 | 说明 |
|------|----------|------|
| API Keys | localStorage | 加密存储（可选） |
| 自定义 Endpoint | localStorage | - |
| 用量统计 | localStorage | 累计数据 |
| 预算设置 | localStorage | 各厂商月度上限 |
| 对话历史 | sessionStorage | 刷新清空 |
| 偏好设置 | localStorage | 模式、语速等 |
