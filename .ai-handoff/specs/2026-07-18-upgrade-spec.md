# 第三方科技 AI 聚合平台 — 功能升级需求指令书

> 目标受众：下游 AI 开发助手（前端开发）
> 项目地址：https://smalluniverseheng.github.io/AI/
> 当前状态：纯前端 PWA，单文件 HTML（或少量 HTML 文件），GitHub Pages + jsDelivr CDN 部署
> 开发环境：黑鲨4pro + Termux（触屏操作，避免 Ctrl 组合键、vim 模式切换）

## 一、项目现状（禁止改动现有逻辑）

当前网站为纯前端 AI 聚合聊天平台，已具备以下功能，**全部保持不动，只做增量添加**：

- **模型库**：270+ 模型，覆盖 23 家厂商（OpenAI、Anthropic、Google、DeepSeek、Kimi、通义千问、智谱、文心一言、腾讯混元、MiniMax、火山引擎、小米 MiMo 等）
- **对话模式**：单模型对话、多模型并行（现有）、辩论模式、协作模式（现有，需升级）
- **输入输出**：文本输入、语音输入（浏览器内置）、语音播报（浏览器内置）、文件上传
- **右侧工具栏**：文本润色、文章摘要、代码解释、角色助手（12 个预设）—— 这些本质是"技能开关"
- **设置面板**：通用（播报/语音识别/主题/语言）、插件与数据（联网搜索 Tavily / 数据管理备份恢复）、帮助、关于
- **主题系统**：亮色 / 暗色 / 跟随系统，颜色变量已定义
- **PWA**：可安装到桌面，离线可用
- **数据存储**：全部 localStorage，支持 JSON 导出备份 / 导入恢复
- **Token 统计**：按厂商与模型分别统计消耗，已有统计页
- **联网搜索**：Tavily 插件，用户自填 Key，前端直接调用
- **翻译空间**：独立页面（正在开发中）

**绝对禁止**：修改现有聊天核心逻辑、模型库数据结构、主题 CSS 变量、API 调用层（除非新增集群模式分支）、设置面板现有选项的行为。

## 二、核心新增功能（三大模块）

### 模块 A：AI 集群协作模式（Swarm / 多 Agent 调度）

**目标**：将现有"多模型并行/协作模式"升级为像 Kimi "K3 集群"那样的智能调度系统——一个组长模型拆解任务、分配给多个组员模型并行执行、回收结果、判断是否继续循环、最终整合输出给用户。

#### A1. 架构要求

```
用户输入
    │
    ▼
┌─────────────────────────────────────────┐
│  Round 0：组长（Orchestrator）分析       │
│  输出：JSON 格式的任务清单（DAG 有向无环图）│
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  执行循环（最多 3 轮，纯前端控制）        │
│  1. 提取无依赖任务 → 并行下发给组员       │
│  2. 等待当前批次全部完成                  │
│  3. 结果汇总 → 发给组长                   │
│  4. 组长判断：完成 / 补充新任务 / 强制终止 │
└─────────────────────────────────────────┘
    │
    ▼
最终答案 → 以"集群工作流卡片"形式展示在对话流中
```

#### A2. 角色与配置

- **组长（Orchestrator）**：1 个，由用户从模型库选择。负责拆解任务、决策循环、整合结果。Prompt 必须约束输出为严格 JSON。
- **组员（Worker）**：3~5 个，每个绑定一个模型 + 一个角色标签（如"搜索员""编码员""分析师"）。
- **预设集群模板**：用户可保存/加载配置，如"编程团队""研究团队""写作团队"。
- **策略参数**：最大轮数（1~5，默认 3）、并行度（受浏览器限制，实际 3~5）、超时时间（默认 30s）。

#### A3. 组长输出 JSON Schema（必须严格可解析）

```json
{
  "analysis": "对需求的简要分析",
  "tasks": [
    {
      "id": 1,
      "assignee": "模型标识（如 deepseek-chat）",
      "role_tag": "搜索员",
      "task_description": "具体任务描述",
      "expected_output": "期望格式（Markdown/JSON/代码）",
      "depends_on": []
    }
  ],
  "integration_prompt": "收到结果后如何整合的指令",
  "should_continue": false,
  "reason": "是否继续循环的原因",
  "final_answer": "如已完成，直接输出最终答案（此时 should_continue 为 false）"
}
```

#### A4. 前端执行引擎

- 纯 JS 实现状态机：`SwarmEngine` 类
- 状态流转：`idle → planning → executing → integrating → completed`
- 浏览器并发限制：同一域名最多 6 个并发，超出自动排队。建议子任务控制在 3~5 个。
- 容错：单个组员失败自动重试 1 次，仍失败标记为 `failed`，结果字符串为 `"[任务失败：原因]"`，由组长在整合时处理。
- 流式输出：组长输出 JSON 时，前端累积完整 chunk 后解析，解析失败则降级为单模型直接回答。

#### A5. UI 呈现（对标 Kimi K3 集群）

**顶部状态栏**：
- 当前单模型模式下显示模型名称（如"Kimi K2"）
- 集群模式下显示"集群名称 + 状态"（如"编程团队 · 执行中"）
- 点击展开**集群配置抽屉**（从顶部下滑）：组长选择、组员列表（可增删改）、策略参数、预设模板切换

**输入框区域**：
- 集群模式下占位符文案变为"给 AI 团队派个活..."
- 底部工具栏：左侧 "+" 按钮（点击展开工具/技能/集群成员浮层）、中间技能标签、右侧语音按钮
- "+" 浮层内容：常用工具（文件/联网/代码）、技能库列表、集群成员快捷管理

**对话流中的集群消息卡片**（核心差异化）：
- 不是普通文本气泡，而是结构化工作流卡片
- 顶部显示："🤖 集群 · 3 个子任务 · 2 轮完成"
- 内部折叠/展开：各子任务卡片（执行者、状态、耗时、查看结果按钮）
- 状态动画：Planning（脉冲"拆解中"）、Executing（进度条/转圈）、Integrating（"整合中"）、Completed（最终答案默认展开）
- 任务卡片默认折叠，最终答案默认展开，用户可点击追溯每步原始输出

**模式切换**：
- 在设置面板或顶部标题栏增加"集群模式"开关
- 关闭时回到现有单模型逻辑，零改动
- 开启时所有相关 UI 切换为集群风格

### 模块 B：插件市场 + 技能库（前端 Mock 架构，预留后端接口）

**目标**：像 Kimi 那样在设置面板和输入框区域集成插件/技能入口，支持安装、配置、开关。当前纯前端阶段用 localStorage 模拟，数据结构预留后端接口字段。

#### B1. 插件（Plugin）定义

插件 = 可调用的外部能力，当前阶段分为三类：
1. **纯前端工具插件**：零 API Key，CDN 引入 JS 库即可（如二维码生成、JSON 格式化、密码生成器、Mermaid 流程图、Markdown 思维导图、番茄钟、文本 Diff、颜色工具箱、时间戳转换、网页截图）
2. **第三方 API 插件**：需用户自申请免费 Key，前端直接 fetch（如天气查询、IP 归属地、汇率换算、GitHub 用户卡、必应壁纸）
3. **联网搜索插件**：已有 Tavily，作为第一个已安装插件的样板

**插件数据结构（预留后端字段）**：
```json
{
  "id": "qrcode-generator",
  "type": "tool",
  "displayName": "二维码生成",
  "category": "效率",
  "icon": "svg-string",
  "description": "文本/URL 转二维码，支持下载 PNG",
  "accessLevel": "free",
  "installed": true,
  "config": {},
  "source": "cdn", // 或 "api" / "builtin"
  "cdnUrl": "https://cdn.jsdelivr.net/npm/qrcode@1.5.3/build/qrcode.min.js",
  "entryFunction": "generateQRCode",
  "permissions": []
}
```

#### B2. 技能（Skill）定义

技能 = 预设 Prompt 模板 + 可选工具组合。当前右侧工具栏的"文本润色、文章摘要、代码解释"就是内置技能。

**技能数据结构**：
```json
{
  "id": "text-polish",
  "name": "文本润色",
  "description": "病句修改、表达升级",
  "promptTemplate": "你是一位资深编辑。请对以下文本进行润色...",
  "requiredTools": [],
  "enabled": true,
  "builtin": true,
  "category": "写作"
}
```

**新增技能库（Prompt 模板，只写 Prompt 即可）**：
- 论文阅读助手、简历优化师、邮件撰写、小红书文案、SQL 生成器、正则生成器、Bug 报告生成、会议纪要整理、产品 PRD 生成、面试模拟、头脑风暴、菜谱生成、旅行规划、财务分析、代码审查、PPT 大纲生成、翻译官（专业版）、降重改写

#### B3. UI 呈现

**设置面板改动（增量）**：
- 现有"插件与数据"区域下方新增两个入口：
  - **插件市场** → 跳转独立页面 `market.html`（或浮层）
  - **技能库** → 跳转独立页面 `skills.html`（或浮层）
- 已安装插件在设置面板中显示配置项（如 Tavily 已有联网搜索配置，新安装的插件同理出现配置卡片）

**插件市场页**：
- 顶部：搜索框 + 分类 Tab（全部/效率/开发/信息/媒体）
- 主体：卡片网格，每个卡片含图标、名称、描述、权限标签（免费/已安装/会员专享）
- 点击卡片：底部弹出半屏抽屉，显示详情 + 安装/配置/升级按钮
- 付费插件：显示"升级解锁"，点击弹出 Toast"后端服务部署后开放支付"

**技能库页**：
- 顶部 Tab：[我的技能] [发现更多]
- 我的技能：已启用的技能卡片，可拖拽排序、编辑、删除（内置技能不可删）
- 发现更多：从常量列表读取，点击"添加"写入 localStorage，自动同步到右侧工具栏
- 自定义技能：底部"+ 自定义技能"按钮，用户输入 Prompt 模板 + 命名，保存后出现在"我的技能"

**右侧工具栏改动**：
- 现有 4 个开关（文本润色/摘要/代码解释/角色助手）改为动态渲染
- 数据源：`localStorage` 中 `skills.enabled` 数组
- 用户从技能库添加新技能后，右侧自动出现对应开关
- 开关状态变化实时影响该轮对话的 System Message 注入

**输入框 "+" 浮层**：
- 分三块：常用工具（文件/联网/代码）、技能库快捷列表、集群成员管理
- 技能库列表显示最近使用的 3 个技能 + "更多"入口

#### B4. 数据层（localStorage 扩展）

扩展现有备份 JSON 结构，新增三个字段：

```json
{
  "version": "2.0",
  "settings": { ... },
  "apiKeys": { ... },
  "conversations": [ ... ],

  "plugins": {
    "installed": ["tavily-search", "qrcode-generator"],
    "configs": {
      "tavily-search": { "apiKey": "tvly-xxx", "enabled": true },
      "qrcode-generator": { "enabled": true }
    }
  },

  "skills": {
    "enabled": ["text-polish", "article-summary", "code-explain"],
    "custom": [
      { "id": "custom-1", "name": "我的润色", "promptTemplate": "...", "category": "写作" }
    ]
  },

  "subscription": {
    "plan": "free",
    "expiresAt": null,
    "searchUsedToday": 12,
    "searchQuota": 50
  },

  "swarm": {
    "templates": [
      {
        "name": "编程团队",
        "orchestrator": "claude-3-5-sonnet",
        "workers": [
          { "model": "deepseek-chat", "role_tag": "编码员" },
          { "model": "kimi-k2", "role_tag": "文档员" }
        ],
        "maxRounds": 3
      }
    ],
    "activeTemplate": "编程团队"
  }
}
```

**备份/恢复**：导出全量备份时包含以上新增字段，恢复时解析并写入 localStorage。

### 模块 C：订阅与会员中心（前端 Mock，预留支付接口）

**目标**：像 Kimi 那样在设置面板中提供"会员计划"入口，展示当前身份、权益对比、升级方案。当前纯前端阶段只做界面和交互，支付链路预留。

#### C1. 角色体系

| 角色 | 标识 | 插件权限 | 技能权限 | 搜索配额 | 存储配额 |
|------|------|----------|----------|----------|----------|
| 访客 | guest | 仅免费插件（只读） | 仅免费技能 | 5次/天 | 0 |
| 普通用户 | user | 免费插件 + 基础付费 | 免费技能 | 50次/天 | 100MB |
| 高级用户 | premium | 全部插件 | 全部技能 | 500次/天 | 1GB |
| 管理员 | admin | 全部 + 管理后台 | 全部 + 编辑 | 无限 | 10GB |

#### C2. 会员中心页

- 当前身份卡片（免费用户/高级用户），显示当前权益清单（✓/✗）
- 升级方案对比：免费版 vs 高级版（¥29/月）vs 企业版（联系定制）
- 权益清单：全部插件、搜索次数/天、自定义技能、存储空间、优先支持
- 支付方式占位：支付宝 / 微信图标，按钮文案"后端服务部署后开放支付"
- 用户点击"立即升级"→弹出 Toast 提示，不跳转外部页面

#### C3. 数据

所有价格、权益、功能对比写死在 JS 常量中。`subscription` 对象存在 localStorage，默认 `plan: "free"`。

## 三、UI/UX 统一规范

### 样式约束
- **颜色**：完全复用现有主题 CSS 变量（`--bg-primary`、`--text-primary`、`--accent-color` 等），禁止引入新色彩体系
- **图标**：使用 Lucide React 或现有图标字体，保持统一风格
- **字体**：复用现有字体栈
- **圆角/阴影**：复用现有设计令牌
- **动画**：集群状态切换用脉冲动画（CSS `@keyframes`），技能安装用淡入淡出，页面切换用滑动过渡

### 移动端优先
- 所有新增页面以移动端布局为基准（max-width 640px）
- 底部固定导航栏（市场/聊天/设置/我的）—— 可选，根据现有布局调整
- 触摸手势：左滑返回、下拉刷新、长按菜单
- 虚拟键盘适配：输入框自动上移，避免遮挡
- 抽屉/浮层：从底部上滑（市场页详情）、从顶部下滑（集群配置）、从左侧滑出（设置面板）

### 对标 Kimi 的交互细节
- 集群消息卡片：默认折叠子任务、展开最终结果，与 Kimi K3 集群一致
- 插件市场：卡片网格 + 底部抽屉详情，与 Kimi "获取应用"一致
- 技能浮层：输入框左侧"+"点击展开垂直列表，与 Kimi 快捷技能浮层一致
- 设置面板：会员计划入口放在"插件与数据"区域下方，与 Kimi 设置层级一致

## 四、技术实现约束

### 绝对禁止
1. 修改现有聊天核心逻辑（消息发送/接收/渲染流）
2. 修改现有模型库数据结构（270+ 模型、23 厂商的列表）
3. 修改现有主题 CSS 变量和切换逻辑
4. 修改现有设置面板中已有选项的行为（如 Tavily 配置、数据备份恢复）
5. 引入需要后端才能跑的核心功能（如真实支付、用户登录体系）
6. 使用需要构建工具（Webpack/Vite）的复杂框架（保持纯 HTML/CSS/JS 或极轻量库）
7. 引入国外平台依赖（Vercel、Clerk、Stripe 等）

### 推荐技术栈
- **前端**：纯 HTML5 + CSS3 + Vanilla JS（ES6+），或极轻量库（如 Alpine.js 3.x，CDN 引入）
- **CDN 资源**：jsDelivr（与现有一致），所有第三方库通过 CDN 引入
- **存储**：localStorage（主），IndexedDB（可选，用于大量插件/技能缓存）
- **API 调用**：复用现有 `fetch` 封装，新增 `callSwarm()` 方法
- **状态管理**：原生 JS Class + localStorage 同步，不引入 Redux/Vuex

### 文件结构建议
```
AI/
├── index.html              # 主站（现有，增量添加集群/技能/插件入口）
├── market.html             # 插件市场（新增独立页面）
├── skills.html             # 技能库（新增独立页面）
├── subscription.html       # 会员中心（新增独立页面）
├── css/
│   ├── theme.css           # 现有主题变量（不动）
│   └── new-components.css  # 新增组件样式（集群卡片、市场卡片、抽屉）
├── js/
│   ├── app.js              # 现有主逻辑（不动）
│   ├── swarm-engine.js     # 集群调度引擎（新增）
│   ├── plugin-manager.js   # 插件管理器（新增）
│   ├── skill-manager.js    # 技能管理器（新增）
│   ├── ui-components.js    # 新增 UI 组件（抽屉、卡片、浮层）（新增）
│   └── storage.js          # 扩展后的 localStorage 读写（新增）
└── manifest.json           # PWA 配置（更新新增页面缓存）
```

### 开发环境适配（Termux/手机端）
- 所有文件编辑用 nano 或手机文件管理器，避免 vim 模式切换
- 预览用 `python -m http.server 8080` 或 `npx serve`
- 部署：GitHub Pages 自动部署，jsDelivr CDN 加速
- 调试：手机浏览器开发者工具（Chrome DevTools Remote）

## 六、验收标准

### 集群模式
- [ ] 用户选择"编程团队"模板，输入"写一个 Python 爬虫并分析其性能"，组长自动拆解为"写代码""性能分析""文档说明"三个子任务

- [ ] 最终答案由组长整合，展示在卡片底部
- [ ] 单模型模式和集群模式可一键切换，切换后 UI 无残留状态

### 插件市场
- [ ] 打开插件市场页，显示分类 Tab 和卡片网格

- [ ] 卸载后配置项消失，localStorage 同步更新
